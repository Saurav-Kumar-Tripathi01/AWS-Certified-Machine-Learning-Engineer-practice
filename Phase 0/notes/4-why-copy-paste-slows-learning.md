# Why Copy‑Paste Slows Learning

## 1. Why This Matters
Copy‑paste gives quick results but creates fragile knowledge.

When things break, you’re stuck.

---

## 2. The Problem This Solves
Without understanding:
- small changes cause failure
- debugging becomes impossible
- confidence collapses

---

## 3. Mental Model
Copy‑paste is like memorizing dance steps
without knowing the rhythm.

You can’t adapt.

---

## 4. Better Alternative
Instead of copying:
- type the code yourself
- change variable names
- break it intentionally
- fix it again

---

## 5. When Copy‑Paste Is OK
✅ After understanding  
✅ For boilerplate  
❌ For learning concepts  

---

## 6. When You’ll See This Again
This rule is critical for:
- ML pipelines
- AWS IAM policies
- Production systems

---

## 7. One‑Line Summary
Understanding creates flexibility; copying creates dependency.
