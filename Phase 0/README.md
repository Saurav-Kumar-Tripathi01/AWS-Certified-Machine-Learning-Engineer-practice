# Phase 0: Developer Mindset Reset

## Phase Objective
Before touching any technology, build the correct *developer mindset*.  
This phase ensures you approach learning like an engineer — not a passive consumer.

Machine Learning, AWS, and systems engineering **will fail you** if this mindset is missing.

---

## Core Beliefs You Must Adopt

✅ Developers learn by building  
✅ Developers break things and fix them  
✅ Developers read error messages carefully  
✅ Developers Google effectively  
✅ Developers improve incrementally, not instantly  

---

## Non‑Negotiable Rules

- Never watch without doing
- Never copy code without understanding it
- Never skip errors — investigate them
- Never memorize blindly — reason it out

---

## Practical Expectations

By the end of this phase, you should:
- Feel comfortable being confused temporarily
- Actively debug instead of panicking
- Treat errors as learning signals

---

## Success Criteria

You are ready to move forward only if:
- You experiment freely without fear of breaking things
- You investigate errors calmly
- You prefer understanding over speed
